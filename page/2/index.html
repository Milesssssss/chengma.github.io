process

2016-06-21

process 对象

process 对象是Node的一个全局对象，提供当前Node进程的信息。
它可以在脚本的任意位置使用，不必通过require命令加载。该对象部署了EventEmitter接口。

进程信息

退出码

进程退出时，会返回一个整数值，表示退出时的状态。

  • 0 正常退出
  • 1 发生未捕获错误
  • 5 v8执行错误
  • 8 不正确的参数
  • 128+信号值。

属性

process对象提供一系列属性，用于返回系统信息

  • process.argv 返回当前进程的命令行参数数组
  • process.env 返回一个对象，成员为当前shell的环境变量。 如process.env.HOME
  • process.installPrefix node安装路径的前缀
  • process.pid 当前进程的进程号
  • process.platform 平台 Linux
  • process.title 默认值为 node
  • process.version Node 的版本

stdout stdin stderr

stdout
  1. stdout
    指向标准输出。
    process.stdout.write(‘message\n’) 等同于console.log
  2. stdin 标准输入
    process.stdin.pipe(process.stdout)
    标准输入导入到标准输出
  3. stderr 指向标准错误

argv, execPath, execArgv

argv: 返回一个数组，由命令行脚本时的各个参数组成。
第一个成员总是node
第二个成员是脚本文件名
其余来自脚本文件的参数

execPath: 返回当前执行脚本的Node二进制文件的绝对路径。

execArgv: 返回一个数组，成员在命令行下执行脚本时，在node可执行文件与脚本文件之间的命令行参数。

process.env

返回一个对象，包含了当前shell的所有环境变量。

process 对象提供的方法

  • process.chdir() 切换工作目录到指定目录
  • process.cwd() 返回当枪脚本的工作目录的路径
  • process.exit() 退出当前进程。
  • process.getgid() 返回当前进程的组id
  • process.getuid() 用户id（数组）
  • process.nextTick() 指定回调函数在当前执行栈的尾部，下一次event loop之前执行
  • process.on() 监听事件
  • process.setgid() 指定当前进程的组
  • process.setuid() 指定当前进程的用户
proess.exit()

退出当前进程
触发exit 事件

process.on()

监听各种事件，并指定回调函数

process 支持的事件有以下yixie

  • data事件： 数据输出输入时触发
  • SIGINT事件： 接收到系统信号时触发

使用时，向该进程发出系统信号，就会导致进程退出
kill -s SIGINT [process_id]

SIGTERM信号表示内核要求当前进程停止，进程可以自行停止，也可以忽略这个信号。

进程不会立马退出，而是执行完回调之后再退出

事件

exit事件

当前进程退出时，触发 exit 事件

回调函数只能执行同步操作，不能包含异步操作，因为执行完回调函数之后，进程立马退出。

beforeExit

在node清空了eventloop之后，再没有任何任务时，给的最后一次机会，使得node进程不退出。

主要区别 exit

可以部署异步操作

如果未发生捕获错误，或者显式终止程序，则不触发beforeExit

uncaughtException

当进程抛出一个未捕获的错误时触发

CSS 权威指南

2016-04-22

CSS 选择器

分组

  • 使用，号来分割，定义一个规则，右边的样式，将应用到左边的所有元素。
  • 通过分组，可以将某些类型压缩在一起。这样可以得到一个更简洁的样式表。
通配符选择器。*
  • 应用到所有元素。

类选择器

要应用样式而不考虑具体是哪些元素，通常使用类选择器。

ID选择器

#id 唯一性。

属性选择器

href，title…

  • 可以多个属性链接在一起使用。
  • 同时可以使用属性值进行精确选择。
子串匹配属性选择器
  • ~=value 值中包含value单词的属性。
  • ^=value 值中以value开始的属性。
  • $=value 值中以value结尾的属性。
  • *=value 值中包含value的属性。
  • |=value 匹配语言值，如[src|="img"] ==> img-1.png。
  • parent > children 匹配子元素。
  • + 匹配相邻兄弟元素。
伪类

静态伪类

  • ：link，：visited
    动态伪类
  • ：focus，：hover，：active
伪元素

*：first-child：第一个子元素的元素。

结合伪类

可以在同一个选择器中结合使用伪类，顺序指定不重要。

值和单位

em和ex

  • em：当前元素字体大小*em值。
  • ex：相对于字符X的高度，通常为字符的一半。

    deg 角度值，解释0~360度范围内的度数，可以是负值。

字体

字体的显示依赖于客户端，如果没有定义的字体，将使用客户端默认的字体样式。

字体系列

Serif字体

  • 字体成比例，而且上下有短线。如果字体字符根据其大小不同有不同的宽度，称为是成比例的。

Sans-serif

  • 字体成比例，而且上下没有短线。

Monospace

  • 字体不成比例的，通常用于模拟打印机打出的字体。，字体宽度完全相同。

Cursive

  • 模仿人的手写字体。主要由曲线和serif字体中没有笔画的字体组成。

Fantasy

  • 这些字体无法用任何特性来表示。只有一点是确认的，就是无法用任意之中类型归类该字体样式。
  • font-family:value value有特殊字符，或者有空格，多个单词，使用引号。
  • 通用字体系列，如果加了引号，浏览器会认为是一种名为该通用字体名的字体。

font-weight

  • 100~900 取值范围。
  • normal 为400。，bolder 为700。
  • lighter，bolder。

font-style

明确italic和oblique文本之间的差别。

  • italic：意大利体。
  • oblique：简单的字体倾斜。
  • normal：
  • inherit：

font-variant

小型大写字母不是一般的大写字母，是采用不同大小的字母。

font-stretch

  • wider：比父元素字体更宽。

文本属性

text-indent 缩进。

作用于块级元素，无法使用到行内元素，不会改变当前盒子宽度。

  • 百分比：相对于当前块。
  • length：具体数值。

text-align

justify：两端对齐，字符宽度由用户浏览器和其他因素决定。如letter-space等。

line-height

最基本的情况是可以增加或减少文本行之间的垂直间隔。

  • 定义文本线之间的最小距离，而不是一个绝对值。文本线拉开的距离可能比line-height值更大。
  • 并不影响替换元素的布局，当时确实可以引用到替换元素中。

letter-spacing
word-spacing

文本转换

text-transform

  • uppercase 转换为大写。
  • lowercase 转换为小写。
  • capitalize 每个首字母大写。
  • 默认值为none，不做任何改动。

文本装饰

text-decoration

  • underline 下划线。
  • overline 顶端划线。
  • line-through 中划线。
  • blink 文本闪烁。

文本阴影

text-shadow

  • 第一个值定义阴影的颜色，第二个，第三个值定义阴影的偏移距离，第四个值定义了阴影的模糊半径。

处理空白符

white-space

  • 会影响用户代理对资源文档中的空格，换行和tab字符的处理。
  • normal：默认情况下浏览器已经完成了空白处理：把所有的空格合并为一个空格处理。
  • pre：空白符不会被忽略。
  • nowrap：不能换行，除非出现br。
  • pre-wrap：文本保留空白，但是文本会正常的换行。
  • pre-line：与pre-wrap相反，文本会合并空白符，但保留换行。

文本方向

direction

  • ltr
  • rtl

基本视觉格式化

基本框

CSS假定每个元素都会生成一个或多个矩形框，这称为元素框。各个元素框中心有一个内容去（content area）。这个内容区有可选的内边距，边框，外边距。
边框与内容和内边距有相同的背景。边框不能为负

使用auto

  • marging-left，margin-right和width 的值可以设为auto。
  • 当其中两个设置为固定值，另外一个设为auto，auto值的宽度会被自动拉伸到夫容器。
  • 当三个都使用固定长度，并且总和小于父容器时，margin-right会被force to be the rest of the length。
  • 当width设置成为auto时，margin-left：auto； margin-right：100px；margin-left会被设置成0；
负外边距
  • 当水平元素过分受限时，有一个规则，要求重置右外边距。这也会得到一个负值。
  • 只有外边距能小于0；

替换元素

  • width为auto时，实际宽度为img的宽度。
  • 如果一个替换元素的width 不同于自身的宽度，那么它的高度也会成比例变化。反过来也一样。

垂直格式化

一个元素的默认高度右其内容决定。高度会随其内容的变化而相应改变。

  • 如果元素的高度设置小于内容的高度，会产生滚动条。overflow属性设置滚动条。
  • 如果margin-top和margin-bottom都设置成为auto，那么他们会被强制设置成为0。也就无法实现垂直居中。但对于定位元素来说将有所不同。
百分数高度

元素设置高度为百分数，为针对父元素的高度的百分比。

  • 如果包裹元素height设置为auto，那么它将于内部元素高度相同。
合并垂直外边距

两个外边距中教小的会被较大的合并。元素的内边距和边框绝对不会合并。

负外边距

负外边距导致元素重叠
元素后出现可能会覆盖较早出现的元素
note：包裹元素width设置成auto，内部负外边距，同样对父元素同级的兄弟元素位置产生影响。

内联元素

img也是内联元素，只不过是替换元素。

行布局
增加框属性

使用padding可以增加内联元素的框的距离，但不影响行高。

  • 替换元素可以增加行高，但不影响line-height。
  • 如果没有line-height值，vertical-align百分比数值就会失效。
  • 为行内替换元素增加border，margin和padding会增加行的高度。
替换元素与基线
  • 替换元素默认下行内距的底端与父元素的基线对齐。

改变元素的现实 display

run-in：将元素定义为run-in框，后面跟着块元素，便会有嵌入后一个元素的效果。并且根据原树等级继承属性。

内边距，边框，外边距

基本元素框

所有文档元素都生成一个矩形框，称为元素框（element box），它描述一个元素在文本布局中所占用的空间大小。因此，每个框影响着其他元素框的位置和大小。

宽度和高度

不能应用到行内非替换元素中。

外边距

百分比使用父元素的宽度的百分比。

浮动和定位

浮动元素

  • 会以某种方式将元素从文档的正常流中删除，不过它还是会影响布局。
  • 这种影响来源于一个现象：一个元素浮动起来，其他元素会环绕该元素。
  • 浮动元素外边距不会合并。

不浮动

float:none

浮动的详细内幕

  • containning block：浮动元素的保含块就是其最近的祖先元素。
  • 浮动元素会生成一个快级框，而不论它本身元素是什么。
  • 浮动元素规则：1.浮动元素外边距不能超越包含框的内边距。2.浮动元素外边距不会覆盖，从合。3-5：不能比之前浮动元素的顶端高。7.如果没有足够的空间，元素会被挤到下一行。

清除浮动

clear确保元素不会出现在浮动元素中。

  • 会增加元素的上外边距，使之出现在浮动元素的下方。
