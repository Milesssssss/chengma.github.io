第一章：轻松搞定CSS

  • 设计模式–精心组织的解决方案

    • 常用功能的组合
    • 模式的名字，要解决的问题，解决方案和各方面的权衡。

  • 1.2 使用设计模式

    • 通过把45种常见的CSs属性与4种元素（内联，内联块状，块状和表格）和5种定位方法（静态，相对，绝对，固定，浮动）结合创建。
    • 强大威力：很容易利用基础的模式并将它们组合来构造出更加复杂的模式。
    • background：CSS内建的设计模式。
    • 绝对定位(absolute)：相对另外一个元素从新定位。
    • 文本替换（Text Replacement）：文本藏在图片的后面。这样一来，图片下载失败，可以显示文字。
    • 左外边距（Left Marginal）：个或多个元素移出块状元素。可以使标题放在左边，内容放在右边。
    • 外边距图片字母下沉（Marginal Graphic Dropcap）：块状元素的左边距里创建一个图形化的首字母下沉，集文本替换和左外边距模式之大成。

  • 1.3 引用样式表

    • 在head区域使用link元素的href属性。

  • 1.4 CSS语法

    • CSS代码应当小写（属性和值不区分大小写）。
    • 常量值不用引号
    • 分号用来终止CSS规则和Import语句。
    • CSS注释 /* CSS */

  • 1.5 使用层叠排序

    • 规则划分为6组(高优先级会覆盖低优先级规则 高->低)
      • 1.使用 ！important 规则群组拥有最高优先级。
      • 2.第二优先级：嵌套在 style 中的样式。由于不便于维护，不建议使用。
      • 3.拥有一个或多个 id 选选择符的规则。
      • 4.拥有一个或多个类，属性，或伪类选择符的规则。
      • 5.拥有一个或多个元素选择符的规则。
      • 6.统配选择符。 e.g. *{border:opx}
    • 位于同一选择符群组并且拥有同样的等级和数量。优先级会按照位置来排。6种位置群组排列顺序（高->低）
      • 1.在HTML文档头部中的 <style> 元素。会覆盖@import导入的属性。
      • 2.<style> 元素中@import 引入的规则要高于 link 引入的规则。
      • 3.通过 link 元素链接的样式表。
      • 4.通过 link 链接的样式表中通过 @import 语句引入的样式表。
      • 5.终端用户绑定的样式表。
      • 6.浏览器所提供的默认样式表。

  • 1.6 简化层叠

    • 为了保持叠层尽可能简单，把绑定的样式最小化，但是不使用 @import 语句。
    • 避免使用 !important。
    • 对每个选择符进行排序。
      • 层叠顺序
      • 通配选择符
      • 元素选择符
      • 类，属性和伪类选择符
      • ID选择符
      • ！important 通赔选择符
      • !important 元素选择符
      • !important 类，属性和伪类选择符
      • ！important ID选择符

  • 1.14 变量的度量单位

    • em：赋值给元素的font-size。在font-size这种情况下，是赋值给它的父类。例如，5em是指5倍的font-size。要把元素相对其文本来设置时，是文档布局依据你的文本大小而伸缩。
    • ex：指元素的当前字体中字母x的高度，与em是相关的，很少使用它。

  • 1.15 固定的度量单位

    • in：逻辑上的英尺。
    • px：像素。对图片定位时很有用，图片是以像素为单位度量的。
    • pt：点。1/72 in
    • pc：指派卡。
    • cm：逻辑厘米。
    • mm：毫米。

  • 1.19 使样式表规范化

    • 载入你的页面将有多块。
      • 寻找你的文件 延时0.1-0.5秒。宽带连接阶段。
      • 为了提高性能，浏览器把文件放在缓存里。
      • 当缓存的文件到期了，浏览器会询问那个服务器文件是否更改。
      • 文件没有到期，即使文件做了修改，浏览器也不会跟新。

第二章：HTML设计模式

- 如何结构化和语义化的使用元素。
- 在设计模式中主要有4种元素：结构化块状元素，终端块状元素，多目标块状元素和内联元素。

  • 2.2 HTML结构

  • 2.7 结构化块状元素

    • 把文档结构化，这样浏览器才能解析出一个更好的文档视图
    • 用块状元素来构造文档，有助于识别它的结构。
    • 4种主要的结构化块状元素：<ol>,<ul>,<dl>和table。
    • 9种支持结构化的元素：<li>,<dt>,<dd>,<caption>,<thead>,<tfoot>,<tbody>,<colgroup>,<col>。

  • 2.8 终端块状元素

    • 需要从文档结构转移到内容
    • <p>,<h1>,<blockquote>,<dt>,<address>和<caption>。
    • 文档块状结构的终端节点。它们不能再包含块状元素。
    • 它们只能包含文本和内联元素。
    • 从结构上来说，它们是其他终端和结构化块状元素的兄弟节点，包含的都是与父元素主题相关的子话题。

  • 2.9 多目标块状元素

    • 需要通过在结构中嵌套结构或终止当前结构来自由扩展文档的结构。
    • <div>,<li>,<dd>,<td>,<th>,<form>和<noscript>。
    • 既可以包含块状元素也可以包含内容，但不能同时包含两种。
    • 块状元素不应该为内联元素和文本的兄弟元素。
    • 应当把内容放在块状元素之中
    • 混合内容的时候会给内容套上一个匿名容器。

  • 2.10 内联元素

    • 需要为文本赋予明确的含义，还想为文本添加样式以反映其内在的含义。
    • 内容与结构相分离。
    • 划分为4类：语义元素，排列顺序元素，可替代元素，控制元素。

  • 2.11 类和ID属性

    • 需要在具有相同元素中识别出某些元素。为一些元素赋予额外的语义和关联。
    • 通常名字全部小写。

  • 2.12 HTML 空白

    • 通过空白提高阅读性，但是不希望空白影响文档的解析。
    • 浏览器会消除同一行中重复的空白。
    • 在连续的空白字符中，第一个空白字符决定了被移除的空白位置与样式。

第三章：CSS选择符与继承

- 主要是用于选择元素，以便添加样式。

  • 3.2 类型，类和ID选择符

    • 通过类型，类和ID选择符，从而赋予他们样式。
    • 尽量用小写命名。

  • 3.3 定位和群组选择符

    • 根据元素的位置把几个选择符组合起来以缩小选择的范围。
    • 子选择符，后代选择符，群组选择符，兄弟选择符，首子选择符等。

  • 3.4 属性选择符

    • 依据元素中是否包含某属性，属性中是否包含某个值，属性中是否设定为某个值。
    • 属性存在选择符，属性词汇选择符。

  • 3.5 伪类元素选择符

    • 需要选择元素的第一个字母或第一行。
    • first-line和first-letter。
    • 使用范围：只与终端块状元素相搭配。对内联元素和结构化块状元素不起作用。

  • 3.6 伪类选择符

    • 根据超链接的状态，来赋予样式。
    • a:link:未被浏览的链接。
    • a:visited:已被浏览的链接。
    • a:hover:鼠标处于悬停状态的链接。
    • a:focus:非IE中获得焦点的链接。
    • a:active:获得在IE中焦点的链接。

  • 3.7 子类选择符

    • 需要用常见的规则为一类元素赋予样式。
    • 你还想为这些元素划分为几个子类，并为他们赋予一定的样式从而覆盖掉基础样式。
    • e.g. class.subclass.etc。

  • 3.8 继承

    • 需要为一个元素赋予样式，并让它的后代元素也应用同样的样式。
    • inherited CSS提供的常量，可以给任何属性，该属性从它的父属性继承值。
    • 模拟继承 selector *{}。

  • 3.9 视觉继承

    • 想让子元素的背景与父元素一样。
    • CSS自动把元素背景设为透明。
    • 如果外边距或者定位引起兄弟元素的重叠，那么后面的兄弟会覆盖住前面的。
    • 浏览器默认把子元素置于父元素的上面的透明层。除非设置了子元素的背景。

第四章：盒模型

- 定义了元素是如何被看做盒子来被解析的。
- 6种盒模型：内联，内联块状，块状，表格，绝对定位和浮动。

  • 4.2 显示

    • 从根本上改变元素的解析方式。
    • 用 display 改变元素的解析方式。
    • display:none 阻止对元素的解析。
    • display:inline 会对元素当作内联进行解析。
    • display:block 或者 display:list-item 把元素当作块状或者列表项进行解析。
    • display:inline-block 把内联元素当作嵌套在块状元素来解析。

  • 4.3 盒模型

    • 想为元素的盒模型赋予样式。
    • 定义了如下属性之间的关系：
      • display
      • width
      • height
      • padding
      • border
      • margin
      • background
      • overflow：溢出
      • visibility：决定元素是否可见
    • 6种主要的盒模型：内联，内联块状，块状，表格，浮动和绝对定位。

  • 4.4 内联盒模型

    • 以内联的排列顺序进行解析的。
    • 内联格式化环境：当超过他们最近的终端块状祖先的宽度时，便换到新的一行。
    • width,height,overflow 在内联上不起作用，因为他们总是与内容的宽度和高度相一致。
    • 边框会覆盖掉临近行

  • 4.5 内联块状盒模型

    • 想把一个内联块状元素赋予样式。
    • 内联块状元素包括替代元素和按内联块状来显示的内联元素。
    • 位于内联排列顺序中，只不过，它还加入了一些块状元素的属性：高度，宽度，外边距，边框，内边距

  • 4.6 块状盒模型

    • 竖直方向从上至下排列。
    • width : 默认值 auto 使得与父元素相一致。
    • height : 默认值 auto 让元素的高度包裹住所有的子元素。
    • margin-left : 如果设置了宽度，margin-left：auto 会让块状元素排列在父元素的右边。
    • margin-right : 如果设置了宽度，margin-left：auto 会让块状元素排列在父元素的左边。

  • 4.7 表格盒模型

    • 表格是含有很多单元格的块状盒模型。
    • 表格有外边距，但是没有内边距。
    • 单元格有内边距，但是没有外边距。

  • 4.8 绝对定位盒模型

    • 从正常的元素顺序中脱离，把它置于更高一层或者更低一层。
    • 相对于最近的定位祖先，或者固定于视窗的某个位置。
    • z-index 控制定位元素的层叠顺序。

  • 4.9 浮动盒模型

    • 使用 float:left 和 float:right 可让任何元素浮动起来。
    • 浮动元素也脱离了正常的元素排列顺序。
    • 被置于临近块状元素的边框和背景之上。这回缩小浮动元素的父元素。

第五章：盒模型的尺寸

  • 5.2 宽度 : width

    • 除了内联元素，width对所有元素起作用。
    • width:auto
      • 水平包裹：内联，内联块状，浮动，表格和绝对定位（当left和right都为auto的时候）。
      • 水平拉伸：块状，绝对定位（left和right都设置成一个值）。

  • 5.3 高度 : height

    • 除了内联元素，height对所有的元素起作用。
    • 浏览器会忽略静态内联元素的height，因为font和font-size绝对了它的文本宽度，从而决定了它的高度。
    • 用height：100%可拉伸表格。

  • 5.4 设定尺寸

    • 带有尺寸的元素必须把width和height设置成百分比。
    • width 和 height 用于定义元素的内盒。
    • 表格是一个例外，它的width和height定义的是表格边框外的尺寸。包括了内边距和边框。
    • 浮动元素设定了尺寸，会改变排列的顺序。
    • 当静态块状元素设定了尺寸，它会改变排列的顺序。
    • 当绝对定位元素被设定尺寸之后，它不会改变原来排序的顺序。也不会改变他的位置。

  • 5.5 包裹

    • 减小元素的宽度和高度从而使其包裹元素的内容。
    • 被包裹的元素需要把元素的width和height设置成auto，从而自动调整和模型的大小。
    • 块状元素不能水平包裹。

  • 5.5 拉伸

    • 把元素高与块拉伸到与容器相同。
    • 通过 auto 100% 对不同类型的盒模型进行拉伸。

第六章：盒模型的属性

  • 6.2 外边距 margin

    • 负值会造成元素的重叠。
    • 对内联元素 margin-top 和 margin-bottom 是被忽略的。
    • 对于大多数元素来说 margin：auto 和 margin：0 是等价的。

  • 6.3 边距 border

    • 运作机理与外边框基本一致。
    • 竖直内联元素边框是不会自动增大行高的。

  • 6.4 内边距 padding

    • 运作机理与边框基本一致。
    • 竖直内联元素边框是不会自动增大行高的。
    • 默认值为0

  • 6.5 背景 background

    • background
      • background-repeat:no-repeat 不平铺
      • background-repeat:repeat 平铺
      • background-repeat:repeat-x x轴平铺
      • background-repeat:repeat-y y轴平铺
      • background-position:水平坐标 竖直坐标

  • 6.6 溢出 overflow

    • 常量：visible，hidden，scroll和auto。默认值为visible
    • CSS3 提供了 overflow-x，overflo-y。

  • 6.7 可见性 visibility

    • 把一个元素隐藏起来，并留下元素所占用的位置。
    • hidden，visible（默认值）

  • 6.8 分页

    • 打印的时候插入分页符。
    • page-break-before 和 page-break-after
    • value：always and auto。

第七章：定位模型

  • 7.2 定位模型

    • 6种定位模型：
      • 静态（static）：对内联，内联块状，块状和表格盒模型进行定位。
      • 绝对和固定（absolute and fixed）：对盒模型进行定位。
      • 浮动定位模型（）：能够对浮动盒模型进行定位。
      • 相对定位模型（relative）：对绝对定位盒模型外的所有盒模型。
      • 相对浮动定位模型（relative-float）：对浮动盒模型进行定位。

  • 7.3 定位

    • static是position的默认值。不对元素进行定位。
    • relative：当前位置进行偏移。
    • absolute：定位最近的祖先进行偏移。
    • fixed：参照视窗进行偏移。

  • 7.4 定位最近的祖先元素

    • 当一个元素没有定位祖先元素的时候，body就是他的祖先元素。
    • 当你对一个自含式元素进行定位时，它的后代元素也会跟随着它移动。
    • 弱点是元素必须被设定大小。

  • 7.5 堆叠环境

    • 对元素堆叠顺序进行控制
    • z-index 值为auto或者赋值给非定位元素时，不参赛堆叠环境。
    • 每种局部堆叠环境都被赋予了一个内部的堆叠等级：0，其后代都是相对于它的。
    • 它是相对于被赋予了z-index值的定位最近的祖先元素而言的。

  • 7.6 原子化

    • 想让文本和内联内容作为块状元素的整体一起被解析，这样，当其他块状元素把它覆盖的时候，它的内容也被覆盖。
    • 定位元素就是原子化的。意味着没有外部元素能够插在它的静态子孙元素，内联内容和背景之间。

  • 7.7 静态定位

    • position：static应用到元素上，可以按照正常的顺序进行定位。这是默认设置。
    • 当内联元素内容超出了容器的宽度时，浏览器会自动把它们移到新的一行。

  • 7.8 绝对定位

    • 想把元素从正常的排列顺序中移除，并把它移到自己的定位层。
    • position：absolute 把任何元素当作绝对盒模型解析。
    • widht 和 height 设置大小，百分比相对最近的盒模型定位吗，而不是他的父元素。

  • 7.9 固定定位

    • 当视窗滚动时，你不想它随之移动。
    • position：fixed 相对视窗而不是相对父元素

  • 7.10 相对定位

    • 控制元素在正常排列位置上的堆叠顺序。
    • position：relative 使用z-index

  • 7.11 浮动和清除

    • 从正常排序中清除，并让他们位于父元素的左侧或右侧。
    • float：left|right

  • 7.11 相对浮动

    • 想在不影响其他元素定位的前提下，把浮动元素从现有的位置上偏移，还有堆叠顺序。
    • 加了position：relative 保留原来的元素位置，用 left 和 top 做一定的偏移。

第八章：定位：缩进，偏移和对齐。

  • 8.2 缩进

    • 把元素拉伸至元素的四边，然后再对它自己的边进行偏移。向内缩进减少元素的尺寸。
    • 向外缩进扩大元素的尺寸。
    • 每一边都可独立的缩进和外凸。
    • 先对齐后缩进。
    • margin 正的缩进 负的外凸。

  • 8.3 静态元素的偏移

    • 对前面元素的结束位置开始计算。
    • 正值向右移动，负值向左移动。

  • 8.4 静态表格的偏移和缩放

  • 8.5 浮动元素的偏移

  • 8.6 绝对元素和固定元素的偏移 position:absolute fixed

  • 8.7 相对元素的偏移 position:relative

    • 通过left right top bottom 移动位置而不影响其他元素位置顺序。

  • 8.8 静态内联元素的对齐

    • position：static
    • text-align ： justify 两端对齐
    • vitical-align ：value 基于基准 向上向下偏移value

  • 8.9 静态块状元素的对齐与偏移

    • margin-right 和 margin-left 都为auto时 中心对齐。

      • sized-static block
      • stretched-static block
      • sized-absolute element
      • shrinkwrapped-absolute element
      • stretched absolute element
